在 VS Code 中，用 AI 帮你高效写代码

1. 什么是 Claude Code

Claude Code 是 Anthropic 公司开发的一个 AI 编程助手。它直接嵌入到 VS Code 中，就像给你配了一个经验丰富的结对程序员，能：

  • 📖 读懂你的代码：理解整个项目的结构和逻辑
  • ✏️ 帮你写代码：新增功能、修复 Bug、重构代码
  • 🔧 执行命令：运行测试、安装依赖、操作 Git
  • 🔗 连接外部工具：Jira、数据库、Slack、Figma 等
  • 🤖 自主完成任务：你描述需求，它自动读代码、写代码、跑测试、提 PR
通俗理解：就像有个老程序员坐在你旁边。你说"帮我加个登录功能"，他就自己去读项目代码、写功能、跑测试、提交 commit，你只需要审查结果。

本指南聚焦：VS Code 扩展方式使用 Claude Code，配合 CC Switch 桌面应用管理多个 AI 账号与 API 提供商。

2. 安装与配置

2.1 安装 VS Code 扩展

在 VS Code 中按 Ctrl+Shift+X 打开扩展面板，搜索 "Claude Code"（发布者为 Anthropic），点击安装。

📌 系统要求

VS Code 版本需 1.98.0 或以上。Windows / macOS / Linux 均支持。

2.2 安装 CC Switch（推荐）

CC Switch 是一个完全免费的开源桌面应用（GitHub 107K+ Stars），用于统一管理和切换多个 AI 工具的账号配置与 API 提供商。它支持 Claude Code、Claude Desktop、Codex、Gemini CLI 等多款 AI 编程工具。

它能做什么

  • 🔀 多账号一键切换：个人 Pro 账号、公司 Team 账号、第三方 API 提供商随意切换
  • 🔗 本地代理路由：拦截 AI 工具的 API 请求，转发到不同的 API 提供商（如 DeepSeek、Kimi、GLM 等）
  • 📊 用量仪表盘：实时追踪 Token 消耗、花费、缓存命中率
  • ☁️ 云同步：通过 WebDAV 或 S3 备份配置，多台电脑同步
  • 📝 会话管理：跨工具查看和恢复对话历史

安装方式

CC Switch 是独立桌面应用，从 GitHub Releases 下载安装：

平台安装方式
Windows下载 .msi（推荐）或便携版 .zip
macOS下载 .dmg，或 brew install --cask cc-switch
Linux.AppImage、.deb 或 .rpm

📦 下载地址：github.com/farion1231/cc-switch/releases

💡 典型场景

白天用公司提供的 API 端点写工作代码，晚上切换到个人 Anthropic 订阅写个人项目。在 CC Switch 里点一下切换，VS Code 中的 Claude Code 自动使用对应的 API 配置，不用改任何配置文件。

2.3 首次登录

  1. 安装 Claude Code 扩展后，VS Code 右侧会出现 Claude Code 面板（或按 Ctrl+Esc 调出）
  2. 首次使用会弹出浏览器让你登录 Claude 账号
  3. 登录成功后即可开始使用

如果你安装了 CC Switch 桌面应用，可以通过它切换不同的 Claude 账号或 API 提供商，每个账号的会话和设置相互独立。

3. 快速上手（5 分钟实践）

🎯 目标

跟着下面的步骤，5 分钟内完成第一次 AI 辅助编程体验。你不需要配置任何东西，装好扩展就能开始。

第 1 步：打开 Claude Code

在 VS Code 中打开你的项目，按 Ctrl+Esc（或点击编辑器右上角的 ✨ 火花图标），Claude Code 面板会出现在右侧。

第 2 步：让 Claude 认识你的项目

在输入框中输入：

帮我分析一下这个项目是做什么的，主要用了哪些技术

Claude 会自己读取项目文件，给你一个概述。这让你快速确认它"看懂"了你的代码。

第 3 步：做一个简单修改

打开任意一个源码文件，选中一段代码，按 Alt+K（Mac: Option+K）插入引用，然后说：

给这段代码加一些注释，解释每个步骤在做什么

Claude 会修改代码。改动以 VS Code 原生 diff 视图展示——左边是原始代码，右边是修改后。你可以逐条 接受 或 拒绝。

第 4 步：让 Claude 帮你查问题

输入：

这个项目里面有没有明显的 bug 或者代码坏味道？帮我检查一下

Claude 会扫描代码，列出发现的问题，并给出修复建议。

第 5 步：用命令完成 Git 操作

如果刚才的修改你满意，输入：

帮我提交这些修改，写一个清晰的 commit message

Claude 会生成 commit message 并提交（首次需要你批准 git commit 命令）。

打开 Claude Code 的三种方式

方式操作适合场景
快捷键Ctrl+Esc最常用，在编辑器和 Claude 间快速切换
火花图标编辑器右上角的 ✨ 图标鼠标操作用户
新标签页Ctrl+Shift+Esc同时进行多个独立对话

界面布局说明

Claude Code 面板通常显示在 VS Code 右侧：

  • 对话区：你和 Claude 的对话内容，支持滚动查看历史
  • 输入框：在底部，输入你的问题或指令，按 Enter 发送
  • 状态栏：显示当前权限模式（default / acceptEdits / plan）、模型名称等
  • 命令菜单：输入 / 查看所有可用命令

会话中的基本操作

操作作用

Esc中断 Claude 正在做的操作——发现它跑偏了立刻按

模型与性能类

/effort [级别]设置努力级别：low / medium / high / xhigh / max
/fast [on|off]切换快速模式
/advisor [model|off]启用/关闭顾问工具（第二个模型给建议）
/thinking切换深度思考模式

上下文管理类

命令作用

/compact [说明]压缩对话历史，释放上下文空间

5. VS Code 扩展详解

5.1 核心功能一览

VS Code 扩展是 Claude Code 的官方原生界面，把 AI 编程助手无缝嵌入你的日常开发环境：

功能说明
✨ 火花图标编辑器右上角，点击打开 Claude Code，当前打开的文件自动成为上下文
@ 文件引用输入 @ 后跟文件名，自动补全路径。如 @src/auth.ts
@ 选区引用选中代码后按 Alt+K，插入带行号的引用如 @app.ts#5-10
原生差异视图Claude 的代码修改在 VS Code 原生 diff 视图中展示，可逐条接受/拒绝
多标签页支持同时打开多个独立对话，互不干扰
会话历史侧边栏可浏览所有历史对话，随时恢复继续
诊断共享IDE Problems 面板的错误和警告自动共享给 Claude
图片支持拖拽、粘贴（Ctrl+V）或文件路径引用图片，Claude 能分析截图和设计稿
PR 状态如果当前分支有开着的 PR，状态栏用颜色标示审查状态

5.2 @ 文件引用详解

这是 VS Code 扩展最重要的交互方式之一。输入 @ 触发自动补全：

  • 模糊匹配：输入 @auth 会匹配所有包含 "auth" 的文件路径
  • 目录引用：@src/components 引用整个目录
  • 选区引用：选中代码后按 Alt+K（Mac: Option+K），自动生成 @文件名#行号 格式的引用

使用示例：

"解释 @src/utils/auth.js 里面的逻辑"
"@src/components 这个目录的结构是什么样的？"
"帮我把 @app.ts#42-58 这段代码重构一下"

5.3 差异视图（Diff View）

Claude 修改代码后，VS Code 会弹出原生的差异对比视图：

  • 左侧是原始代码，右侧是 Claude 修改后的代码
  • 绿色高亮 = 新增，红色高亮 = 删除，蓝色高亮 = 修改
  • 你可以逐条 接受 (Accept) 或 拒绝 (Reject) 每个改动

5.4 多标签页会话

VS Code 扩展支持同时打开多个 Claude Code 标签页，每个标签页是一个独立的对话：

  • Ctrl+Shift+Esc 打开新标签页
  • Ctrl+Shift+T 重新打开刚关闭的会话
  • 每个标签页有独立的上下文和历史

5.5 选中代码发送

在编辑器中选中代码，右键菜单选 "发送到 Claude"，或直接按 Ctrl+Esc 打开 Claude 面板，选中的代码会自动作为上下文。

5.6 配合 CC Switch 使用

CC Switch 在后台运行一个本地代理服务，自动拦截 Claude Code 的 API 请求，路由到你当前选中的账号/提供商。你在 CC Switch 桌面应用中切换账号后，VS Code 里的 Claude Code 会立即使用新的配置，无需重启。

  • 多账号的会话历史、设置、记忆、权限配置相互独立
  • 不同账号可以使用不同的 API 端点（Anthropic 官方、第三方代理、私有部署等）
  • 用量仪表盘可以查看每个账号的 Token 消耗和花费

6. 记忆系统

Claude Code 有两套记忆机制，让它在不同会话间"记住"你的偏好和项目规范。

6.1 CLAUDE.md 文件

这是你手动编写的 Markdown 文件，Claude 每次启动会话都会读取。就像给 Claude 的"项目说明书"。

层级位置用途
用户全局~/.claude/CLAUDE.md个人偏好，所有项目通用
项目共享项目根目录/CLAUDE.md 或 .claude/CLAUDE.md团队共享的项目规范（可提交 git）
个人本地项目根目录/CLAUDE.local.md你个人对这个项目的偏好（应加入 .gitignore）

怎么写：

  • 保持在 200 行以内（越长越容易被忽略）
  • 要具体："使用 2 空格缩进" 而非 "格式写对"
  • 包含：构建命令、编码规范、架构决策、测试要求、常见坑
  • 不要写：Claude 读代码就能看出来的东西、标准约定
  • 运行 /init 让 Claude 自动分析项目并生成初始版本

6.2 自动记忆

Claude 在工作过程中自动写给自己的笔记。它会自动记录构建命令、调试心得、架构认知、代码风格等信息，在后续会话中自动参考。

通过 /memory 命令可以开启或关闭自动记忆。所有记忆文件都是纯文本 Markdown，你随时可以查看和编辑。

6.3 .claude/rules/ 规则目录

大型项目可以把规范拆成分类文件，放在 .claude/rules/ 目录下：

project/
  .claude/
    CLAUDE.md          ← 主项目说明
    rules/
      code-style.md    ← 代码风格
      testing.md       ← 测试规范
      security.md      ← 安全要求

还可以用 YAML 前置元数据限定规则只对特定文件生效：

---
paths:
  - "src/api/**/*.ts"
---
# API 开发规范
- 所有端点必须包含输入验证

7. 设置与配置

Claude Code 采用层级覆盖机制来管理配置，不同位置的配置文件有不同的优先级。

7.1 配置层级（从低到高）

层级位置影响范围
用户~/.claude/你的所有项目
项目.claude/（可提交 git）所有项目成员
本地.claude/settings.local.json（gitignore）只有你、这个项目

7.2 配置文件对照表

配置内容用户位置项目位置
设置~/.claude/settings.json.claude/settings.json
子代理~/.claude/agents/.claude/agents/
MCP 服务器~/.claude.json.mcp.json
CLAUDE.md~/.claude/CLAUDE.mdCLAUDE.md
快捷键~/.claude/keybindings.json无

7.3 常用设置项

  • 模型配置：model（默认模型）、fallbackModel（备用模型）、effortLevel（努力级别）
  • 行为：autoCompactEnabled（自动压缩上下文）、autoMemoryEnabled（自动记忆开关）、outputStyle（输出风格）
  • 权限：permissions.allow（允许列表）、permissions.deny（拒绝列表）、permissions.defaultMode（默认模式）

使用 /config 命令在会话中直接修改设置，例如 /config model=claude-sonnet-4-6。

7.4 环境变量配置（API 端点与 Token 认证）

如果你需要通过自定义 API 端点（比如代理、私有部署）来使用 Claude Code，而不是默认连接 Anthropic 官方 API，你需要在配置文件中设置环境变量。

配置方式

在 .claude/settings.json（或 ~/.claude/settings.json 全局配置）中添加 env 字段：

{
  "env": {
    "ANTHROPIC_BASE_URL": "https://你的API端点地址",
    "ANTHROPIC_AUTH_TOKEN": "你的认证Token"
  }
}

这两个变量分别是什么

变量名作用示例值
ANTHROPIC_BASE_URL指定 Claude API 的基础地址。如果你用的是代理、中转服务或私有部署，这里填对应的 URLhttps://api.example.com
ANTHROPIC_AUTH_TOKENAPI 认证 Token。如果不使用浏览器 OAuth 登录，而是用 Token 直接认证，就填在这里sk-ant-xxxx

常见使用场景

  • 🔗 API 代理/中转：公司或团队搭建了统一的 API 网关，所有请求通过代理转发
  • 🏢 私有部署：使用 Anthropic 的私有云部署端点
  • 🔑 Token 认证：使用 API Key 而非浏览器登录（如 CI/CD 环境、无头服务器等）

完整配置示例

一个典型的 settings.local.json（不提交 git，只对你生效）：

{
  "env": {
    "ANTHROPIC_BASE_URL": "https://your-proxy.example.com",
    "ANTHROPIC_AUTH_TOKEN": "sk-ant-api03-your-token-here"
  },
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test *)"
    ]
  }
}

⚠️ 安全提醒

千万不要把包含真实 Token 的 settings.json 提交到 git 仓库！建议把 env 配置放在 .claude/settings.local.json（自动被 gitignore），或者放在用户全局配置 ~/.claude/settings.json 中。

💡 配合 CC Switch

如果你用了 CC Switch 管理多个账号/API 提供商，每个提供商可以在 CC Switch 中配置各自的 URL 和 Token。切换提供商后，Claude Code 自动使用对应的端点，不需要手动改 settings.json。

怎么确认配置生效

配置完后，在 Claude Code 中输入 /status，查看显示的信息中是否使用了你配置的端点地址。如果配置不正确，Claude Code 会提示连接失败。

7.5 实战：使用 DeepSeek V4 模型

Claude Code 支持通过自定义 API 端点接入第三方模型。以下介绍如何通过 CC Switch 或直接配置 settings.json 的方式，让 Claude Code 使用 DeepSeek V4 模型。

方式一：通过 CC Switch 添加 DeepSeek V4 Pro API

  1. 打开 CC Switch 桌面应用
  2. 点击左侧 "提供商管理" 或 "Add Provider"
  3. 填写以下信息：
    • 名称：DeepSeek V4（自定义，方便识别）
    • API 端点：https://api.deepseek.com（或你的 DeepSeek Pro API 地址）
    • API Key：你的 DeepSeek API Key（在 DeepSeek 平台获取）
  4. 保存后，在 CC Switch 主界面切换到 DeepSeek V4 提供商
  5. VS Code 中的 Claude Code 会自动使用 DeepSeek V4 的 API 端点，无需重启

方式二：直接在 VS Code 中配置 settings.json

如果你不使用 CC Switch，也可以直接在 Claude Code 的设置文件中配置。打开 .claude/settings.local.json（项目级，不提交 git）或 ~/.claude/settings.json（全局），添加环境变量：

{
  "env": {
    "ANTHROPIC_BASE_URL": "https://api.deepseek.com",
    "ANTHROPIC_AUTH_TOKEN": "your-deepseek-api-key"
  }
}

参数说明：

参数填什么示例
ANTHROPIC_BASE_URLDeepSeek V4 的 API 地址https://api.deepseek.com
ANTHROPIC_AUTH_TOKEN你在 DeepSeek 平台申请的 API Keysk-xxxxxxxxxxxxxxxx

配置完成后验证

  1. 在 VS Code 中打开 Claude Code 面板（Ctrl+Esc）
  2. 输入 /status 查看当前连接的 API 端点是否已切换为 DeepSeek 地址
  3. 随便问一个问题，确认能正常响应

💡 DeepSeek V4 的优势

DeepSeek V4 是一款高性能国产大模型，支持超长上下文、代码生成能力强，且 API 价格相对低廉。适合个人开发者在 Claude Code 中作为日常编码模型使用。

⚠️ 注意事项

  • DeepSeek API Key 不要提交到 git 仓库，建议使用 settings.local.json 或全局配置
  • 不同第三方 API 提供商的兼容性可能不同，如遇到问题可切换回 Anthropic 官方 API 对比排查
  • CC Switch 方式切换更方便，支持一键在多个提供商之间来回切换

8. 权限系统

Claude Code 对不同操作有不同的安全级别，保护你的代码和数据安全。

8.1 默认安全级别

工具类型例子默认行为
只读读文件、搜索代码不需要批准
命令执行运行 npm、git 等命令需要批准
文件修改编辑、写入文件需要批准

8.2 三种权限规则

  • Allow（允许）：自动批准，不弹窗问你
  • Ask（询问）：每次都弹窗让你确认
  • Deny（拒绝）：直接阻止，绝对不执行

8.3 权限模式

按 Shift+Tab 循环切换模式，状态栏会显示当前模式：

模式自动批准适合场景
default只读上手阶段、敏感项目
acceptEdits只读 + 文件编辑正常写代码（推荐日常使用）
plan只读，不可编辑先探索再动手
auto几乎所有（有安全检查）长时间自主任务

8.4 配置白名单/黑名单

在 .claude/settings.json 中配置：

{
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test *)",
      "Bash(git status)"
    ],
    "deny": [
      "Bash(curl *)",
      "Read(./.env*)",
      "Read(./secrets/**)"
    ]
  }
}

⚠️ 受保护路径

.git、.claude、.vscode、.gitconfig、.bashrc、.envrc 等敏感文件和目录永远不能自动写入。

10. MCP 服务器

MCP（Model Context Protocol，模型上下文协议）是一个开放标准，让 Claude Code 能连接各种外部工具和数据源。

10.1 什么场景需要用 MCP

  • 📄 读取 Google Drive 中的设计文档
  • 🎫 自动更新 Jira 工单状态
  • 🗄️ 直接查询 PostgreSQL / MySQL 数据库
  • 🎨 集成 Figma 设计稿，对照设计写 UI
  • 💬 发送 Slack 消息通知
  • 📊 对接 Sentry、Datadog 等监控工具排查问题

10.2 配置方式

在项目根目录创建 .mcp.json 文件（可以提交 git，团队成员共享），或者用 /mcp 命令在会话中管理。

有三种作用范围：

范围配置文件谁可见
本地.mcp.json（项目根目录）当前项目，只有你的机器
项目.mcp.json（提交到 git）团队所有成员
用户~/.claude.json你所有项目通用

💡 新手提示

刚开始用建议先不配置 MCP，Claude Code 本身的功能已经足够强大。当你需要让 Claude 直接操作数据库或更新 Jira 工单时，再来配置 MCP。

11. Hooks 钩子系统

钩子（Hooks）是在 Claude Code 执行特定操作时自动触发的自定义脚本。适合高级用户做自动化定制。

11.1 常见使用场景

  • 每次 Claude 执行 Bash 命令前做安全检查
  • 每次 Claude 修改完代码后自动跑测试
  • 收到通知时弹出桌面提醒
  • 禁止 Claude 执行某些危险操作

11.2 主要触发时机

时机说明
SessionStart会话开始时触发
UserPromptSubmit你提交问题前触发（可以阻止）
PreToolUse工具执行前触发（最关键——可以拦截、修改输入）
PostToolUse工具执行成功后触发
StopClaude 回复完后触发

📌 新手注意

Hooks 是进阶功能，刚上手时不需要配置。等用熟了，有自动化需求时再回头看。

12. Git 集成

Claude Code 深度集成 Git，你可以在对话中用自然语言操作版本控制：

"帮我提交修改，写一个清晰的 commit message"
"创建一个新分支 feature/quickstart"
"显示最近 5 条 commit"
"帮我解决合并冲突"
"创建一个 PR"

12.1 PR 关联

创建 PR 后，Claude Code 会自动把当前会话关联到这个 PR。当你回到这个会话时，Claude 知道对应的 PR 是什么，可以直接继续工作。

12.2 差异查看

输入 /diff 打开交互式差异查看器，VS Code 原生 diff 视图中显示所有未提交的修改，方便你在提交前检查。

12.3 PR 审查状态

如果你当前分支有打开的 PR，VS Code 底部状态栏会显示 PR 的审查状态（绿色 = 已批准，黄色 = 待处理，红色 = 要求修改，灰色 = 草稿）。

13. Plan 计划模式

计划模式让 Claude 只调研、写计划、不动代码。适合你不确定怎么改、需要先理清思路的场景。

13.1 进入计划模式

  • 按 Shift+Tab 直到状态栏显示 "plan mode"
  • 或在输入框输入 /plan 我想加一个 OAuth 登录功能

13.2 计划完成后

Claude 写完计划后会问你怎么做：

  • 批准，全自动执行 — Claude 自己改代码、跑测试
  • 批准，文件编辑自动，其余手动确认
  • 批准，每条修改都让我审核 — 最安全
  • 继续打磨计划 — 你觉得方案不够好，让它改

💡 什么时候用计划模式？

要用的场景：涉及多文件的大改动、不熟悉要改的代码、不确定最佳方案。
不需要的场景：改个错别字、加一行日志、修一个已知的小 bug。

14. 子代理与工作流

14.1 什么是子代理

子代理是专门的 AI 助手，在自己独立的上下文窗口中运行。你可以把一些费时的探索、分析工作交给子代理，它不会占用主对话的上下文空间。

为什么需要子代理：

  • 🔒 隔离上下文：让子代理去读大量文件做研究，不影响主对话
  • 🎯 专注特定任务：可以给子代理专门的指令和工具权限
  • 💰 控制成本：简单任务可以用便宜模型（如 Haiku）

14.2 内置子代理

  • Explore — 搜索代码库，回答代码问题
  • Plan — 创建详细的实现计划

14.3 使用子代理

"用一个子代理去研究我们的认证系统怎么处理 token 刷新的"
"用子代理审查这段代码有没有边界情况"

14.4 Fork 分叉

/fork <指令> 生成一个继承当前对话的子代理去干活，你可以继续做别的事，它完成后结果返回给你。

14.5 Batch 批量处理

/batch <指令> 把一个大改动拆成 5-30 个独立单元，每个在一个隔离环境里并行实现。

15. 会话管理

15.1 什么是会话

每次你和 Claude Code 的对话就是一个会话。会话会自动保存，你可以随时恢复之前的对话继续未完成的工作。

15.2 在 VS Code 中管理会话

VS Code 扩展提供了图形化的会话管理：

  • 侧边栏历史：左侧面板可以看到所有历史会话，点击恢复
  • 搜索会话：按关键词搜索之前的对话
  • 重命名：给会话起个好名字，方便以后找（选中后按 Ctrl+R）
  • 多标签页：同时打开多个会话标签页处理不同任务

15.3 上下文管理

随着对话变长，上下文窗口会逐渐被占满。几个管理技巧：

命令作用
/clear开始新对话，旧会话保留在历史中可随时恢复
/compact让 Claude 压缩对话历史，总结关键信息释放空间
/context可视化查看上下文窗口占用了多少
/rewind回退代码和对话到之前的检查点
/branch 名称复制当前对话到新分支（原对话保留，尝试新方向）

15.4 后台任务

/background [任务] 把当前耗时任务放到后台，你可以继续其他工作。完成后会通知你。

16. 新手最佳实践

1. 始终给 Claude 一个验证方法 🎯

这是最重要的建议。告诉 Claude 怎么验证自己写对了：

"写一个 validateEmail 函数。
测试用例：user@example.com 应该返回 true，invalid 应该返回 false。
写完跑一下测试，确保通过。"

2. 先探索，再做计划，最后写代码 📋

  1. 探索："先读一下 /src/auth 目录，理解我们的认证逻辑"
  2. 计划：进计划模式，"/plan 我想加 Google OAuth，要改哪些文件？"
  3. 实现：退出计划模式，"按你的计划实现 OAuth 流程"
  4. 提交："写个清晰的 commit message 并提 PR"

小任务直接干，不用每次都计划。

3. 需求要具体 ✍️

  • ❌ "修一下 bug"
  • ✅ "修复登录页的 bug：用户输错密码后页面变空白，应该显示错误提示。相关文件在 @src/Login.tsx"

4. 善用 @ 引用 📎

输入 @ 加文件名，比描述文件在哪快得多。选代码后 Alt+K 直接带行号引用。给 Claude 精确的上下文是得到好结果的关键。

5. 积极管理上下文 🧹

  • 不相关的任务之间 /clear 一下
  • 临时小问题用 /btw，不占主对话空间
  • 大型代码探索交给子代理
  • 对话太长时用 /compact 压缩一下

6. 写好 CLAUDE.md 📝

运行 /init 自动生成，然后逐步打磨。保持在 200 行以内。只写 Claude 读代码看不出来的东西（构建命令、项目规范、常见坑）。

7. 建立权限白名单 ✅

第十次批准 npm run lint 时你应该已经烦了。在设置里加到 allow 列表：

{
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test *)",
      "Bash(git status)"
    ]
  }
}

8. 从粗到细提问 🔍

接触新代码库时，按这个顺序问：

  1. "给我一个这个项目的整体概览"
  2. "讲解一下主要的架构模式"
  3. "认证是怎么处理的？"
  4. "找到处理用户认证的文件，帮我加个 token 刷新"

9. 及时纠偏 🛑

发现 Claude 跑偏了，立刻按 Esc 中断。好结果是靠紧密的反馈循环，不是完美的首次尝试。

10. 避免常见错误 ⚠️

错误症状解决方法
大杂烩会话一个会话里做一堆不相关的事/clear 隔开
反复纠正同样问题纠正 3 次以上/clear + 重写更清楚的提示
CLAUDE.md 太啰嗦Claude 好像不听话砍到 200 行以内
信了就过代码看着对但边界情况漏了始终提供验证方式
无限探索"研究一下 X" 读了上百文件限定范围或交给子代理
不设白名单每次操作都要点批准把安全操作加入 allow

11. 用好状态栏和 CC Switch 📊

  • 运行 /statusline 自定义状态栏，显示上下文用量、花费、模型信息
  • CC Switch 桌面应用管理多账号/多提供商，工作/个人项目一键切换互不干扰

总结

Claude Code 是你在 VS Code 里的 AI 副驾驶——帮你读代码、写功能、修 Bug、做审查。你用自然语言指挥它，它帮你执行。

对于新手，记住这五步：

  1. 装好扩展 → VS Code 商店搜 "Claude Code"，装好即用
  2. 先读代码 → 让 Claude 帮你理解项目结构和逻辑
  3. 小改起手 → 从修 typo、加注释开始建立信任
  4. 逐步放手 → 熟悉后让它自己写功能、跑测试、提 PR
  5. 及时中断 → 发现不对按 Esc，纠偏后继续

方向盘始终在你手里，Claude 是帮你踩油门的副驾驶。
